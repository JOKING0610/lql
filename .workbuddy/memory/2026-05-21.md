## 用户中心路由配置

用户要求：
- 主应用 → 用户中心：`/lql/usercenter`（不是 `/lql/user-center.html`）
- 用户中心 → 主应用：`/lql/`（不是 `/lql/index.html`）
- 用户中心内容使用原来的 `user-center.html`

### 实现方案
- **Dev server**：自定义 Vite 插件 `userCenterPlugin()`，在 `configureServer` 中用 `enforce: 'pre'` + 中间件拦截 `/usercenter` 请求，直接返回 `user-center.html` 的内容
- **Build/生产**：`npm run build` 后自动将 `user-center.html` 复制为 `dist/usercenter/index.html`，GitHub Pages 可正确路由

### 改动文件清单
1. **vite.config.js** — `userCenterPlugin` 插件 + `appType: 'spa'`（SPA fallback）
2. **src/App.jsx** — 2 处导航链接 → `import.meta.env.BASE_URL + 'usercenter'`
3. **src/lib/app.js** — `openLoginPage()` → `./usercenter`
4. **src/pages/user-center/components/sidebar/Footer.jsx** — 返回首页 → `../`
5. **package.json** — build 脚本添加后处理（复制 user-center.html → dist/usercenter/index.html）
6. **test-user-center.html** — 更新测试链接
7. **src/main.jsx** — 移除 React 路由判断，直接渲染 `<App />`

### 关键技术点
- Vite `configureServer` 返回函数时，自定义中间件在内置 SPA fallback 之前执行
- `enforce: 'pre'` 确保插件在其他插件之前注册
- Vite 内部会去掉 base 前缀，所以中间件匹配的是 `/usercenter` 而非 `/lql/usercenter`

### 验证结果
- `/lql/usercenter` → 零柒零音乐 - 用户中心 ✅
- `/lql/` → 零柒零音乐0.3 ✅
- `dist/usercenter/index.html` (195KB) 正确生成 ✅

**JS 逻辑：**
