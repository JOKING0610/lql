## 图片编辑器功能

### 需求
- 上传头像前可进行旋转、裁剪等编辑
- 确保输出比例为 1:1

### 实现
- 新增 `avatarEditorModal` 模态框，包含：
  - Canvas 预览区域（1:1 比例）
  - 圆形裁剪框（SVG 遮罩）
  - 工具栏：左旋转、右旋转、缩放滑块、重置按钮
- 新增 CSS 样式：编辑器布局、工具栏按钮、缩放滑块等

### JavaScript 状态管理
- `editorImage` — 当前编辑的图片对象
- `editorRotation` — 旋转角度（0/90/180/270）
- `editorZoom` — 缩放比例（1 = 100%）
- `editorOffsetX/Y` — 拖拽偏移量
- `window._croppedAvatarBlob` — 裁剪后的 Blob 数据

### 核心函数
- `openAvatarEditor(dataUrl)` — 打开编辑器加载图片
- `closeAvatarEditor()` — 关闭编辑器清理状态
- `renderEditorCanvas()` — 渲染 Canvas（应用旋转、缩放、偏移）
- `rotateEditorImage(deg)` — 旋转图片
- `setEditorZoom(val)` — 设置缩放
- `resetEditorTransform()` — 重置所有变换
- `confirmEditorCrop()` — 确认裁剪，输出 400x400 WebP Blob

### 交互流程
1. 选择非 GIF 文件 → 打开图片编辑器
2. 拖拽/触摸移动图片位置
3. 旋转/缩放调整
4. 点击"确认裁剪" → 生成 1:1 圆形裁剪的 WebP Blob
5. 裁剪结果存储到 `window._croppedAvatarBlob`
6. 点击"确认更换" → 优先上传裁剪后的 Blob

### 特殊处理
- GIF 文件不支持编辑，直接预览
- 编辑器支持触摸手势（移动端）
- 每次打开模态框清除 `window._croppedAvatarBlob`

## 用户中心数据同步 & 跳转重复拉取修复 (2025-07-12 续)

### 问题
1. 用户中心修改数据（头像、播放历史、主题等）只存 localStorage，不上传服务器
2. 跳转回主应用时 `autoLogin()` → `fetchUserData()` 从服务器拉取，会覆盖用户中心的修改

### 修复方案

#### 1. 用户中心新增 `syncUserDataToServer()` 函数
- 使用 `upsert_user_sync_data_v2` RPC 上传 settings 和 plays 数据
- 支持防抖（1500ms）和立即模式
- 上传成功后设置 `dataJustSynced` 时间戳标记

#### 2. 在数据变更处调用同步
- `applyAvatarChange()` — 头像变更后同步
- `confirmClearHistory()` — 清空播放历史后同步
- `handleLogin()` — 登录成功后同步（上传迁移的游客设置）
- `handleRegister()` — 注册成功后同步
- `toggleThemeMode()` — 主题切换后同步

#### 3. 导航标记机制
- `goBack()` 和 `goToApp()` 调用 `navigateToMainApp()` 设置 `dataJustSynced` 标记
- 标记值为 `Date.now()` 时间戳

#### 4. 主应用 `autoLogin()` 检测标记
- 检查 `dataJustSynced` 标记是否在 10 秒内
- 如果是，跳过 `fetchUserData()` 调用，直接使用本地数据
- 清除标记避免影响后续加载
